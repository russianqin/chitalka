# 不切换页面、不打断阅读：我的俄语阅读器是怎么用 AI 帮我记单词的

## 写在前面

学俄语这件事，我已经坚持了快两年。

从最初的字母表开始，到现在已经能磕磕绊绊地读完一些简单的俄语原著。这个过程谈不上轻松，但也不算痛苦——直到我遇到了一个所有语言学习者都会遇到的老大难问题：

**生词。**

---

## 一、痛点：阅读时被生词打断是什么体验

读原版书的时候，遇到生词是家常便饭。一个动词可能变位了，一个名词可能变格了，你不确定它的原型是什么，更不确定它在句子里的精确含义。

这时候，标准的解决方案是什么？

打开词典 App。输入单词。查到这个词。切换回阅读页面。继续读。

这个过程可能只需要 30 秒。但你知道最可怕的是什么吗？

**不是 30 秒本身，而是被打断的阅读心流。**

读原版书最珍贵的状态，是完全沉浸在文本里，跟随作者的文字进入另一个世界。一旦被词典 App 打断，这个状态就消失了。你需要花额外的精力重新进入阅读状态。

更糟糕的是，有些词典 App 还会推送广告、推送通知——等你回过神来，可能已经过了五分钟。

另一个常见方案是 Anki。

Anki 是一个基于间隔重复的记忆工具，非常科学。但它有一个致命问题：**脱离语境。**

你记忆的是孤立的单词，没有上下文，没有例句，没有情感。你记住了"рабо·та = 工作"这个对应关系，但你不知道这个词在"Вчера я ходил на рабо·ту"这个句子里用的是第几格、什么含义。

结果是：单词记住了，用的时候还是不会。

**我需要的是：在阅读原地完成词汇积累。不切换页面，不打断思路，不脱离语境。**

这就是我开发 Читалка Pro 的初心。

### 2.2 发现新问题：变位变格搞乱生词本

解决了自动收录的问题后，新的麻烦来了。

俄语是一门形态丰富的语言。一个动词可能有几十种变位形式，一个名词有单复数六个格。如果我在一本书里遇到 выучил（学习，过去时），又遇到 выучить（学习，原形），再遇到 учу（学习，我学）——

**这三个词会被当成三个不同的单词收录。**

一个月下来，我的生词本里有 300 个词条，但实际上可能只有 100 个不同的原型。大量的重复词条让生词本变得臃肿而混乱。

怎么办？

我的解决方案是：让 AI 自动识别单词的**原型**（lemma）。

当用户点击 выучил 时，AI 会自动识别出这是动词 выучить（学习）的过去时变位形式，然后在生词本里记录的是原型 выучить，而不是 выучил。

为了帮助学习者正确发音，我还添加了**重音标注**功能。俄语的重音不规则，有些词重音位置不同，意思完全不同。比如 рабо·та（工作）和 рабо´та（喧哗），重音不同，含义迥异。通过在元音字母前加中点（·）来标注重音位置，比如 рабо·та。

```
原型（含重音标注）：
рабо·та

英文释义：work / job
中文释义：n. 工作，劳动；[只用单数] 作业，功课
词根词缀：рабо-（词根）+ -та（名词后缀）
记忆技巧：与英语"work"同源，可联想"work at"的工作状态
原文例句：Вчера я ходил на рабо·ту в центр.
语法标注：阴性名词，单数第六格为「на рабо·те」
```

**【用户截图占位②：AI 收录弹窗，7 项字段界面】**

### 2.3 大文本性能危机

随着阅读量的增加，我开始尝试更长的文本——陀思妥耶夫斯基、契诃夫、帕斯捷尔纳克。

问题来了。

当生词本积累到 200+ 个词条时，每次保存一个新词，页面都会明显卡顿。

原因是什么呢？

当时的实现方式是：每次保存生词，就遍历整个文档的每一个俄语单词，把所有生词重新高亮一遍。对于一篇 5000 词的文章，这意味着每次操作都要处理 5000 个 DOM 节点。

在性能分析里，我看到了一个长长的红色火焰图。

解决方案有两个：

**第一，用 Set 数据结构代替数组。**

原来生词查找是 O(V) 的复杂度——每次都要遍历整个生词数组。现在改用 Set 存储生词，查找复杂度降为 O(1)。不管生词本有 100 个词还是 1000 个词，查找速度都一样快。

**第二，增量高亮代替全量重建。**

原来保存一个生词，会把所有生词全部重新高亮一遍。现在只高亮新添加的那个词，其他词不受影响。

这两个优化叠加，效果是显著的。一篇 10000 词的文章，即使生词本有 500 个词，保存新词也不会有任何卡顿感。

```
性能对比（10000 词文章，500 个生词）：

优化前：每次保存 300-500ms，全量 DOM 重建
优化后：每次保存 <10ms，增量更新

用户感受：从"明显卡顿"到"完全无感"
```

### 2.4 翻译需求的觉醒

阅读过程中，我遇到了另一种情况：不是单词不认识，而是整句话看不懂。

比如：

> Погружённый в свои мысли, он вышел из комнаты, не заметив, что дверь осталась открытой.

这句话里每个词我可能都认识，但组合在一起，我就是无法理解它的准确含义。

这时候，我不需要把它们收录到生词本——我只需要一个**翻译**。

但现有的工具无法满足这个需求：

- 词典 App：需要手动输入整个句子，太麻烦
- 翻译 App：需要切换页面，打断阅读
- Google 翻译网页版：同上

我的解决方案是：**选中多词，直接翻译，不保存。**

```
┌──────────────────────────────────────────┐
│  🌐  AI 翻译                        [✕]  │
├──────────────────────────────────────────┤
│                                          │
│  原文（俄语）                             │
│  ┌──────────────────────────────────────┐ │
│  │ Погружённый в свои мысли...         │ │
│  └──────────────────────────────────────┘ │
│                                          │
│  中文翻译                                 │
│  ┌──────────────────────────────────────┐ │
│  │ 他沉浸在自己的思绪中，走出房间时，      │ │
│  │ 没有注意到门还开着。                    │ │
│  │ 这句话描绘了一个心事重重的知识分子形象。 │ │
│  └──────────────────────────────────────┘ │
│                                          │
│            [ 关闭 ]                      │
└──────────────────────────────────────────┘
```

逻辑很简单：

- 点击**单个单词** → AI 收录（保存到生词本）
- 选中**两个及以上单词** → AI 翻译（直接展示译文，不保存）

阅读过程中遇到看不懂的句子，选中、翻译、继续读。整个过程不需要离开阅读页面，不需要保存任何东西。

### 2.5 生词本去重的困扰

随着使用深入，我发现了一个恼人的问题：同一个词的重复收录。

比如在一本书里，作者可能用同一个词 выучить 出现了十几次。每次遇到，我都习惯性地点击收录。结果生词本里就有十几个 выучить 的词条——只是 savedAt（保存时间）不同。

虽然它们都是同一个原型 выучить，但因为是同一次阅读中分批收录的，我没有办法简单地识别和合并它们。

我需要一个**自动去重**的机制。

在 v2.9 版本中，我添加了书签自动去重功能：

**每次添加书签时，自动扫描生词本。对同一原型的多个词条，仅保留最后添加的那一条，其余自动删除。**

这个逻辑很简单：

```
书签添加时执行去重：
1. 按 lemma 分组
2. 每组按 savedAt 倒序
3. 保留最新的一条，删除其余
4. 刷新生词本、高亮、统计
```

```
┌──────────────────────────────────────────┐
│  🔖 书签                           [✕]   │
├──────────────────────────────────────────┤
│  ┌──────────────────────────────────────┐ │
│  │ 《Преступление и наказание.txt》     │ │
│  │ 📅 2026-05-01  📍 14:23:05  📊 67% │ │
│  │                        [跳转] [🗑]   │ │
│  └──────────────────────────────────────┘ │
│  ┌──────────────────────────────────────┐ │
│  │ 《Человек в футляре.md》              │ │
│  │ 📅 2026-04-29  📍 09:15:42  📊 23% │ │
│  │                        [跳转] [🗑]   │ │
│  └──────────────────────────────────────┘ │
│                                          │
│  [➕ 添加书签]  ← 添加时自动去重          │
└──────────────────────────────────────────┘
```

去重完成后，Toast 会提示清理了多少个重复词条：

> 书签已添加，并清理了 3 个重复原型生词

## 四、核心功能详解

### 4.1 阅读器

**拖放即读。**

支持 .txt、.text、.md 等纯文本文件。把文件拖到页面上，或者点击上传按钮选择文件，都行。

**支持超大文件。**

曾经用它读过《战争与和平》的俄语原版，100 多万字符，浏览器毫无压力。内容全部存在内存里，不需要任何后端依赖。

**俄语单词高亮。**

这是我最引以为豪的功能之一。生词本里的单词，在正文中会以浅绿色背景高亮显示。

当你读到一段文字时，一眼就能看到哪些词是你已经收录的（高亮），哪些词是陌生的（正常颜色）。这种视觉反馈让阅读过程更有节奏感——遇到陌生词会特别警觉，遇到熟悉词会有成就感。

**阅读控制栏。**

顶部工具栏提供丰富的阅读控制：

| 控件 | 功能 |
|------|------|
| 字体切换 | 衬线体 / 无衬线体 |
| 字号滑块 | 14px ~ 28px，拖动实时预览 |
| 行距滑块 | 1.2 ~ 2.5，步进 0.1 |
| 加粗按钮 | 一键切换正文加粗 |
| 书签按钮 | 打开书签管理面板 |
| 字数统计 | 显示文档总词数 |

所有排版设置都会自动保存，刷新页面后保持。

### 4.2 AI 智能收录单词

这是工具的核心功能。

**操作流程：**

1. 阅读时点击任意俄语单词
2. 弹出浮动工具栏，显示当前单词
3. 点击「🤖 AI 收录」按钮
4. 弹出收录面板，DeepSeek AI 并行生成 7 项学习资料
5. 用户可编辑所有字段，确认后保存

**AI 生成的 7 项内容：**

| 字段 | 说明 |
|------|------|
| 原型（词典形式） | 动词变位 / 名词变格后的原形，含重音标注（ рабо·та） |
| 英文释义 | 单词原型对应的英文翻译，简洁，1~3 个词 |
| 中文释义 | 精确释义，含语境义 |
| 词根词缀分析 | 前缀 / 词根 / 后缀逐层拆解 |
| 记忆技巧 | 联想记忆、同源词提示 |
| 原文例句 | 从文档中提取的完整句子 |
| 语法标注 | 标注词性、体、时、数、格等 |

**设置项：**

工具需要配置 DeepSeek API Key（需要自行注册 https://platform.deepseek.com）。

API Key 存储在浏览器本地，不会上传到任何服务器。支持的模型包括 deepseek-chat（推荐）和 deepseek-coder。

### 4.3 多词翻译

**操作流程：**

1. 用鼠标选中一段俄语文本（≥2 个词）
2. 弹出浮动工具栏，显示「🌐 翻译」按钮
3. 点击后调用 AI 翻译
4. 直接展示中文译文
5. 无需保存，专注当前阅读理解

这个功能完美满足"整句不懂，需要翻译"的需求，而且不会污染生词本。

### 4.4 生词本管理

**列表模式：**

- 显示所有已收录单词，按收录时间倒序
- 顶部搜索框支持按单词、释义全文搜索
- 点击任意卡片展开详情

**详情面板：**

- 显示完整学习资料（释义、词根、记忆技巧、例句、语法）
- 可手动编辑或删除该词条

**去重机制（v2.9）：**

每次添加书签时，自动清理同一原型的重复词条，仅保留最新添加的一条。

### 4.5 书签功能

**添加书签：**

- 点击阅读工具栏的「🔖 书签」按钮
- 点击「➕ 添加书签」记录当前阅读位置
- 自动记录：添加日期（YYYY-MM-DD）、精确到秒的时间戳（HH:mm:ss）、当前阅读进度百分比

**书签列表：**

- 显示所有书签：文件名、添加日期、添加时间、阅读进度百分比
- 点击任意书签条目跳转至对应位置
- 点击删除按钮移除书签

### 4.6 数据导入导出

**导入：**

- 支持 .csv 和 .tsv 文件
- 自动识别分隔符（逗号 / 制表符）
- 智能匹配列名（识别"单词/слово/word"、"释义/trans"等常见列名）
- 导入前显示预览，确认后执行

**导出：**

点击「💾 备份」按钮，弹出导出面板：

| 格式 | 说明 |
|------|------|
| TXT（文本） | 可选日期范围，包含完整字段，复制到剪贴板 |
| JSON 备份 | 可按日期筛选，完整恢复 |

## 五、技术细节

**单文件 HTML，零安装。**

整个工具就是一个 .html 文件，双击即开。Windows、Mac、Linux 都可用，不需要任何服务器。

**浏览器 localStorage，数据不上传。**

所有数据（生词本、设置、书签）都存在浏览器本地。API Key 也只存在本地，不会上传到任何服务器。

**DeepSeek API 提供 AI 能力。**

AI 收录和翻译功能依赖 DeepSeek 的语言模型。在国内访问稳定，速度快，成本低。

**增量高亮 + Set 哈希，大文本无卡顿。**

即使生词本有几百个词，即使文章有几万词，性能依然流畅。
